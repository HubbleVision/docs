# Text2SQL (NL→SQL and Chart Generation)

This page is an overview for the Text2SQL category. It consolidates endpoints and request/response shapes. Examples are provided on each endpoint page.

## Basics
- Base URL: `https://api.hubble-rpc.xyz/agent/api/v1`
- Authentication: `HUBBLE-API-KEY: <your-key>` (Header, case-sensitive)

<a id="request-response-variants"></a>
## Request/Response shapes
- Authentication: every endpoint requires the `HUBBLE-API-KEY: <your-key>` header.
- Request body: POST endpoints use `application/json`.
- Response types:
  - Non-streaming JSON: returns the full JSON at once.
  - Streaming SSE: returns data as Server-Sent Events. Common event types:
    - `workflow_start`, `node_execution`, `plan_created`, `step_completed`, `llm_start`, `token`, `llm_end`, `node_error`, `result`, `complete`, `error`
    - Chart-related: `chart_start`, `chart_sql_complete`, `chart_generation_start`, `chart_generation_complete`, `chart_complete`, `chart_error`

## Endpoints
- `/status`: health check (GET)
- `/text2sql`: NL → SQL → execute → result (POST)
- `/generate-chart`: NL → SQL → execute → auto charting → chart generation (POST)

> For detailed parameters and examples, open the specific endpoint pages from the left sidebar.
